初看到 Windows Phone 的字眼，是一個微涼的午後，秋風帶着窗外和煦的陽光灑向室內，匆匆關掉了不擅長的遊戲，一條有關微軟的 RSS 映入眼簾。那時的我，還是諾基亞與塞班的擁躉，雖說對於新生的 Android 保持着好奇，但 Milestone 粗製劣造的做工和緩慢的觸控體驗實在難以讓我對它的未來提起興趣。

我就像如今的學生黨那樣，手裏攥着積攢了一個假期的零花錢，覺着買到了心愛的手機，就有了全世界，而現在看來尚還稚嫩的心裏，滿滿地被諾基亞 N8 所佔據。而 Windows Phone 就只是可以被忽略的“新聞”罷了。

可命運從來都會和我們開一些玩笑，比如在隨後的一年時間裏，囊中空空的我沒有買到心心念唸的 N8，也和想要買的 HTC Desire HD 失之交臂，意外用上了代號爲“Schubert”的 HTC HD7，這是我的第一款 Windows Phone 手機，也改變了我的人生軌跡。

一晃眼 7 年過去，代表着 Windows Phone 的 Joe Belfiore 無奈發推，宣佈微軟將不會再爲 Windows 的手機業務提供新的硬件以及功能，這意味着微軟最終沒有逃過 7 年之癢，將 Windows 系列手機永遠定格在 2017 年末的秋天。

“好景不會每日常在，天梯不可只往上爬”，這是謝安琪《喜帖街》的歌詞，卻也貼切這個故事中讀出苦澀的回味——畢竟最好的時光過去了，我們終究要面對現實。

美好暢想的 Windows Phone，曾有着最現代化的理念

2010 年登場的 Windows Phone 作爲 Windows Mobile 的後續版本，在推出之始，微軟就提出了三大設想：世界性的切換動畫、流暢的直覺式交互和排除無用干擾的高速信息流傳遞，而這三大設想無疑在 7.0 版本中全部得以體現。

首先是世界性的切換動畫和流暢的直覺式交互，在 Windows Phone 橫空出世的 2010 年末和 2011 年初，Windows Phone 在高通 8250 單核處理器的帶動下，流暢的滑動和幀數足夠的動畫足夠讓當時處於遠古時期的 Android 衆機感到驚歎。事實上還不僅僅如此，Windows Phone 平臺早期的流暢也體現在一些細節的交互處理上，比如點按按鈕的按壓回饋、輸入鍵盤點擊時系統發出的聲響反饋都頗有劃時代的意義。

Windows Phone 本身也以信息流傳遞爲核心，作爲這一設想的體現，微軟將 Metro 擺在了 UI 設計的主要地位。Metro 設計首先出現於微軟 Kin 系列手機以及 Zune 播放器，其風格採用大量大字體的 Titie，以信息本身爲主題，微軟認爲這種風格的主體應該體現“光滑、快和現代”等元素。這也致使 Windows Phone 本身的風格化與主流的 iOS、Android 完全不同。

這一設計的有點在於統一以及低廉的學習成本，微軟希望信息在 Windows Phone 中通過 Metro 界面能夠簡潔清晰、快速的、並以內容爲主的方式進行展現。以此爲基礎，微軟在第一個版本中設計了 People、Music&Video、Picture、Games、Marketplace 和 Office 六個不同的組件，這些組件以全景視圖的方式在平面上水平切換，它首次提供了一種全新的視圖，在水平方向上擴展內容到屏幕之外，來展現不同的控件、數據和服務，就好像把它們排布在一張橫軸的畫卷上。通過這一系列規範，早期 Windows Phone 的應用程序不僅在 UI 上取得了統一，更在交互上以滑動做到了一致，用戶在使用符合 Metro 規範的應用時幾乎不在需要花費過多的學習成本，統一的 Application Bar 幾乎已經完成了直覺式交互的目的。

但這僅僅是系統層面 WP 的配套服務處處體現着倉促

雖然有着完整的 UI 設計理念，但 Windows Phone7.0 版本登場之時，卻並沒有讓大多數用戶感到滿意，在系統以外的各方面，處處體現着微軟的倉促。首先，在編寫 Windows Phone7.0 版本的初期，微軟即便仍然使用 CE 爲核心，但卻廢除了 Windows Mobile 的大量代碼，這導致 Windows Phone 與 Windows Mobile 的應用無法通用，在 APP 呈現爆炸式增長的 2010-2011 年，號稱十萬應用程序無所不能的 APP Store 已經成爲 iPhone 的殺手鐗，谷歌旗下的應用商店也已經初出茅廬，而 Windows Phone 本身卻要從頭開始。

其次，微軟爲了安全考量，也沒有開放應用的文件讀寫，傳輸音樂、圖片以及視頻只能依靠緩慢的 Zune 客戶端，這本身就爲新用戶設立了一道不小的門檻，要知道，作爲 Zune 播放器連接傳輸的客戶端，Zune 在最初並沒有提供完整的手機文件管理功能，因此各種報錯的錯誤代碼就成爲了老用戶最常見到的故障。直到 Windows Phone8.0 開啓 MTP 連接功能，Zune 客戶端壽終正寢爲止，它的連接管理功能都不能算好用。

然而這些問題和下一個相比，卻都不算是太大的問題。Windows Phone 7.0 初期版本僅提供了英文語言的支持，反而成爲了用戶使用的最大障礙，除了不支持系統不支持中文語言顯示外，Windows Phone7.0 也沒有提供第三方輸入法接口，這意味着用戶本身無法切換中文輸入法鍵盤。因此，在早期的 Windows Phone 的中文應用中，都存有內置中文輸入鍵盤的作法，只有這樣才能讓國內用戶正常用上應用。

而直到 2011 年下半年代號 Mango 的 Windows Phone7.5 推送後，這一問題才真正得到解決。但此時，用戶卻發現，缺乏原生代碼編寫的 Windows Phone 無法令第三方應用保持穩定的流暢度，卡頓的感覺幾乎出現在每一個第三方應用，這與流暢至極的系統表現，形成諷刺的對比。

最好的版本 Windows Phone8 登場 但它卻是滿手鮮血

隨着時間過去，時間進入 2012 年，得益於諾基亞的青睞，此時國內用戶對於 Windows Phone 關注度逐步升溫，因而當 6 月新版本公佈時，國內微軟生態的開發者一面倒的表示了興趣。

現在回望來看，配合 Windows 8 的推出，Windows Phone8 相較 7.0 以及 7.5 版本的確有諸多改進，首先它的內核由 Windows CE 變爲了與 Windows 同步的 NT6.2，這爲未來通用應用的登場打下了技術。基於相同的 UI 設計語言，Windows Phone 8 提供了 VB、C、C++、C#、XAML 以及 HTML5 等語言開發的支持，開發者本身只需要改動很少的細節，即可將 Windows 的應用移植至手機。

其次，Windows Phone8.0 版本還首次對多核心 CPU 提供了支持，手機分辨率的標準也提升至 WVGA、WXGA 和 720P，這使得 Windows Phone 硬件落後的情況成爲了歷史，自此之後，搭載 Snapdragon S4 的 Lumia 920 以及高通驍龍 800 的 Lumia 1520 成爲了 WP 歷史上的經典，它們的硬件水準也與行業保持了近乎相同的水準。同時，它也提供了呼聲較高的 USB 連接模式，整體應用的流暢度也有了較大的提升。

但在 Windows Phone8.0 的背後，卻是老用戶的一片哀鴻，由於內核的改變，因此此前熱門的 Lumia800、HTC Mozart 等設備都無法正常升級至 Windows Phone8.0，針對這些產品，微軟只推送了象徵性的 7.8 版本作爲補償，但它僅僅是裝載有新桌面的 7.5 而已，可以說 Windows Phone8.0 的推出，幾乎是踩在了老用戶血淋淋的屍體上，才一步步的走到了前臺。

外界的意興闌珊 微軟的固步自封 最終讓 WP 在小衆的泥潭中無法自拔

2013 年，微軟宣佈以 80 億美元的價格收購諾基亞智能手機業務，幾乎是在同一時間，Windows Phone 取得了誕生以來最好的市場表現，得益於諾基亞 Lumia 家族在歐洲市場的積累，Windows Phone 在美洲市場一度迫近 6%，而歐洲更是奪得 10%的市場，可這樣的熱潮並沒有真正爲 Windows Phone 業務帶來起色。

回頭來看 Windows Phone 本身的銷售結構，的確滿滿的隱憂。在諾基亞所售出的千萬銷量中，低價產品佔領了主流，Lumia 520/521 開始出現在各大領域，這是一款定位入門的 Lumia 產品，在發售後，擁有低廉售價的 Lumia520 在谷歌的搜索量迅速超過旗艦 Lumia 920，而根據 AdDuplex 公佈的調查數據，Lumia 520 在最鼎盛的時期，佔據了 Windows Phone8 平臺 27%的份額，在所有 Windows Phone 也有 20%的佔有率。而與之相對的，卻是 Lumia 高端產品的遇冷。

低價用戶本身對價格敏感，卻很難真正產生對於系統和體驗的依賴，因此當 2014 年 Android 千元機浪潮開啓後，Lumia 的銷量開始出現雪崩式的下滑，而隨後出場的 Lumia 620、630 等產品也難以重現 Lumia520/521 的盛況。

與此同時，微軟內部也正經歷着對於 Windows Phone 存在意義的討論，期間涉及的人員波動，以及數位 Windows Phone 團隊成員的出走，都對 WP 的發展造成了影響。這樣混亂的局面直至 2014 年 Satya Nadella 上臺後，纔得到了緩解，而這位以雲業務起家的印度裔 CEO 明顯不滿意老鮑爾默那富有激情的一套，在外界看來，清理沒有盈利能力的業務將會是他上任後的第一步，而 Windows Phone 鐵定將會出現在名單中。

很難說究竟是哪個問題造成了 Windows Phone 當前的局面，只能說，在 2014 年之後，整個 Windows Phone 進入了一個難以挽回的惡性循環。雖然微軟團隊仍然極力說服應用平臺推出適用於 WP 平臺的應用，但面對着數以億計的 Android 以及 iOS 用戶，WP 的確很難成爲行業的首選，這意味着 Windows Phone 本身仍舊遭遇着第三方應用的缺失，而持續的應用缺失又將遭遇普通用戶的叛逃，用戶的流失則成爲開發者拒絕爲 WP 開發應用的理由。

因此，總結來看，Windows Phone 本身就像是一羣狂熱的微軟團隊，本着出色理念卻固執怪癖搭建的危牆，而應用開發者、媒體和用戶，以及對手們卻成爲了強大的推手，伴隨着微軟自己一把將牆推下，最終結果只能是轟然倒塌。

但仔細想來，這真不是一個令人感到憂傷的故事，因爲現在我們真的也不再需要 Android 和 iOS 以外的手機系統了。
